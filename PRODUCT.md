# Product

## Register

product

## Users

Developers who run terminal-based coding agents on macOS and want ambient,
at-a-glance lifecycle status without terminal capture, cloud services, or a
separate computer. They primarily interact with the app briefly from the menu
bar, use the simulator while configuring or diagnosing the system, and may later
connect a Waveshare RP2040-Matrix over USB-C.

## Product Purpose

Agent Matrix turns privacy-minimal Codex lifecycle events into seven clear visual
states on a simulated or physical 5x5 RGB matrix. Success means a developer can
install the app, trust the hook explicitly, validate the complete workflow in the
simulator, connect hardware later, and understand working, needs-input, finished,
disconnected, and error states at a glance.

## Brand Personality

Quiet, precise, and trustworthy. The app should feel like a focused native macOS
utility: compact during normal use, transparent when diagnosing behavior, and
expressive only on the matrix where motion conveys real system state.

## Anti-references

Do not resemble a marketing dashboard, electronics hobby project control panel,
or decorative RGB utility. Avoid oversized headings, ornamental cards, novelty
controls, unexplained technical jargon in the primary workflow, and color-only
state communication. Never imply that hooks are trusted, hardware is connected,
or an agent recovered until the corresponding system event confirms it.

## Design Principles

1. Show system truth. Connection, trust, and agent states reflect confirmed events.
2. Keep the common path compact. Normal operation belongs in a scannable menu-bar popover.
3. Reveal diagnostics progressively. Detailed protocol and fault controls live in the simulator and developer settings.
4. Make hardware optional. Every meaningful software workflow works in the simulator first.
5. Preserve user control. Configuration changes are explicit, reversible, and narrowly scoped.

## Accessibility & Inclusion

Use native controls, system typography, keyboard navigation, VoiceOver labels,
and sufficient contrast. Matrix states must differ by shape and motion as well as
color. Respect Reduce Motion by selecting stable representative frames while
preserving semantic state, and support increased contrast without losing status
meaning.
